Convert Decimal to Binary

To start, let's use the number 60 as an example. To convert 60 into binary, you must first keep in mind the values of binary. As a reminder (I highly suggest you read the binary page first before reading this. You'll know what's happening here if you do.), a binary number has 8 bits. I would suggest working from left to right for this. Let's start with this binary number: 00000000

Currently the above is all 0's. That will change when we figure out which ones will become a 1. Here's how I would convert decimal to binary. First I would take the first bit (decimal 128) and see if the number is bigger than it. Since 60 is less than 128, the first bit (starting from the left) will remain a 0. Next is 64. 60 is less than 64 so that remains a 0. Nothing has changed so far.

Now we go on to the next one, which is 32, and here's where the real conversion starts. 60 is greater than 32 so you change that bit to a 1 and subtract 32 from 60 which equals 28.

Next bit is 16. 28 is greater than 16 so set that bit to a 1 and subtract 16 from 28 to get 12.

Next is 8. 12 is greater than 8 so set that bit to 1 and subtract 8 from 12 to get 4.

The next bit is 4 and this is ultimately the last bit we need to set to 1 because the remainder is 4. 4 - 4 = 0. The binary number ends up looking like this: 00111100

I hope that wasn't too confusing. Basically, in this method, you would work with the binary string from left to right and subtract as you go, making the original number smaller and smaller until it becomes 0. This is easier if you start with writing out 8 0's then gradually replacing the 0's with 1's as they fit according to the calculation.

Convert Binary to Decimal

Now that you've learn how to convert decimal to binary, it's time to reverse it! In my opinion, this is slightly easier than converting decimal to binary. All you do is simply look at the binary number and add accordingly. You can add in either direction, you'll get the same result regardless. If the binary bit is a 0, move onto the next bit. If the bit is a 1, add the bits value (according to its position in the string) and continue going until you've added all the bits that contain a 1.

Let's put that into play. We have the binary string 00111100 and we need to reverse the conversion. I'm going to convert this from left to right just because I feel like doing it that way; you can convert it from right to left and get the same result. The first bit is a 0 so we move onto the next bit. The next one is also a 0 so we keep moving on. The 3rd bit from the left is a 1. Its value is 32 so we add that to our calculator. Next is also a 1 and this one is 16 so we add that to 32 (32 + 16). Next bit is a 1 and that one is 8 so we add that to our calculation (32 + 16 + 8). The next bit is the last one that has a 1 and that one is 4 so we add that to our calculation (32 + 16 + 8 + 4) and when you hit enter, you end up with 60. Easy, right? See, that wasn't that hard. I think it's easier to convert binary to decimal than it is to convert decimal to binary.
