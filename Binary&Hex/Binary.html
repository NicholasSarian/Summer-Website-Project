Binary

Binary is essentially the language in which computers use to do almost everything. An example of what a binary number may look like is this: 10011101.

Before we start understanding what this strange number is, you must first understand what a byte and bit is. A byte consist of 8 bits. A bit is basically a single number. For example, 1 is a single bit; 25 is 2 bits, 123 is 3 bits, etc. A binary number usually consist of 8 bits.

Usually, numbers range from 0-9; however, in binary, there are only 2 numbers: 0 and 1. The numerical value of a binary string (a string, in binary, is a single set of 8 bits) depends on where the 1's and 0's are located. In a binary string, the string is read from right to left and the count starts at 1. The 1st bit is 1, multiply that by 2 to get 2. Multiply that by 2 to get 4. Keep on multiplying my 2 until you reach the 8th bit.

Now to understand how to count binary. Basically, based on the binary example I gave above, the count will look like this:

How to Count Binary
1 0 0 1 1 1 0 1
128 64 32 16 8 4 2 1

Basically as you can see in the bottom row, the numbers are getting doubled. Remember, read this from right to left, NOT from left to right. 1 x 2 = 2, 2 x 2 = 4, 4 x 2 = 8, etc.
In binary, to find out what number the string represents, you add the numbers together such as 1 + 128. But hold on there, don't start adding just yet. There's a reason why some of the values on the top row are 1 and some are 0. You can think of the 1's as "on" and 0's as "off"; like flipping a light switch. A 1 means you will count that number in your math problem. A 0 means you ignore the number it represents. In the example above, the math problem would be 1 + 4 + 8 + 16 + 128 which equals 157.

At this point, hopefully you understand how to count binary. It requires very basic math skills. We, as humans, have no use for binary but computers use binary all the time. It's their language and it can help a little to understand it. You don't need to worry about knowing how to actually use binary, it's just useful to understand a little more about binary.
