Terminology About

Bandwidth

Bandwidth is the size of a pathway on a cable. Think of bandwidth as a pipe and water that travels through it as data. The wider the pipe, the more water can flow through it. Thinner pipes allow less water to travel through it than a thicker pipe. In bandwidth, more bandwidth allows more data to travel accross the cable. More is better.

APiPA

APiPA stands for Automatic Private iP Address. APiPA is an address that starts with 168.254.x.x (the "x" is a value that can range anywhere from 0 - 254). Your computer can't use this address to connect to the internet. Your computer will get an APiPA address if it is not able to contact the DHCP server to get a new address or renew the lease (the amount of time your computer will be allowed to use a particular address).

Cache Memory

There is main memory and cache (pronounced as "cash") memory. Main memory is like your hard drive. It stores all your main things. It's bigger but also slower. Cache memory is smaller but faster. Basically, cache memory stores information for the purpose of being able to retrieve the same information quicker in the future. Let's say you constantly visit a site. When you visit the site the first time, it may take a while. Your computer will store the information it's getting from the website into cache memory so that if you visit the site again, it can easily pull that information out of cache memory and make the page quicker to load.

Subnet Mask

The Subnet Mask works alongside with IP Address. The subnet mask is used to determine which part of the IP address is part of the network and which is part of the host ID.

Default Gateway

The Default Gateway is a router that allows you to send packets to other networks. This comes into play a lot when you're going to websites on the internet. Since each website is on a different network, you need someway to exit your network and go to theirs. That's where the default gateway comes into play.

PING

PING stands for Packet INternet Gopher. You would run this in Command Prompt and this tool helps you determine if you're able to connect to a particular host, website, etc.; anything with an IP address. This tests connectivity by sending out 4 (or sometimes 5) packets to the target's name or IP address and if the packet gets there successfully, the target will send replies. These packets are just useless packets that don't have any purpose in itself. In Windows, the packet's payload will contain the alphabet, up to W. If the packet isn't big enough, the alphabet will repeat until the packet becomes big enough to send out. A ping command would look something like this: "Ping 8.8.8.8". The 8.8.8.8 is an IP address (this is actually the IP address of Google's DNS server)
