DNS

DNS stands for Domain Name Service. DNS is responsible for name resolution. Whenever you type in an address like www.google.com, it's the DNS's job to resolve that name and find the IP address that belongs to www.google.com. You already know that, to get to a website you want to visit, you must type the website's address in the address bar. However, this doesn't tell the servers anything about what the IP address of the website you just entered is. DNS basically answers the question "What is the IP address of this host?".

How DNS Works

After every website address you type in is followed by an invisible dot. DNS resolves names backwards; from right to left. DNS starts with that imaginary dot at the end of the address and ask these servers called the Root Servers. There are a total of 13 root servers throughout the world and these root servers are essentially the servers everyone goes to when they need to resolve a name. Each root server consists of a network of multiple servers (because there are billions of people in the world and even more devices). The root server will tell DNS where they can start, which is the .com, .net, .gov, etc. at the end of the address. That extension is called the top level domain. In our example, it ends with .com so the root servers will direct the DNS server to the .com top level domain. That top level domain will direct DNS to google; the second level domain. From there, DNS knows that www. is a computer name so they can start finding the IP address. Once the DNS server has found the IP address, they report back to you the IP address they have found and your computer can use that address to finally connect. Seems like a lot of work, doesn't it? Well, that's what has to happen to get the address of names.

Before starting any DNS lookup, your computer will look at its cache memory (basically a "memory bank" that holds several things, in this case, we're talking about IP address and names) and see if you have previously went to the website. If the website's IP address is already in your computers cache memory, then your computer doesn't have to bother the DNS server and can immediately start connecting to the site.
