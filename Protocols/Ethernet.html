Ethernet

Ethernet is commonly used in a Local Area Network (LAN). A LAN is a network of computers and other electronic devices that span a small area such as an office, building floor, or an entire building. Ethernet is a protocol that controls how data is transmitted over a LAN. Some common speeds of Ethernet is 10BaseT (10Mbps, which is slow), 100BaseT (100Mbps, faster than 10BaseT), and 1000BaseT (1Gbps, this one is relatively common). 100BaseT is called Fast Ethernet and 1000BaseT is called Gigabit Ethernet. These are the speed in which data can cross the cable. If you have a desktop computer, you most likely already have an Ethernet cable attached to the back of your computer. That cable is the cable that your packets are travelling across.

When it comes to sending out packets, there are 3 types of Ethernet addresses:

  • Unicast - The user sends the packet to a single target, i.g. The user sends packets to a website they want to connect to.
  • Multicast - User sends a packet to multiple targets, i.g. Sending an email to multiple people.
  • Broadcast - User sends the packet to everyone on the network, i.g. When the user sends out a notification to everyone in the LAN letting them know about something very important.

In a packet, the source address will always be unicast. There will always be one sender. The Destination address, however, can be unicast, multicast, or broadcast.

The first two fields are relatively self-explanatory. Below is a brief explaination of each field:

Destination Address - This is the address of the target of whom the packet is meant to go to. This section is 6 bytes.

Source Address - This is the address of the sender who sent the packet. This section is 6 bytes.

Type Field - This lets the computer know a brief summary of what it can expect to see in the data field (aka payload). This field is 2 bytes.

Data & Padding - Also known as the Payload, this field carries the main information of the packet. This would carry things like information to load a website, confidential info from a government website, etc. This field's size can range from 46 - 1500 bytes. If this field is less than 46 bytes, then there is padding (empty data) added into the payload to make sure that the packet meets the minimum packet length. An Ethernet frame must be at least 64 bytes long in order to be sent out. This is to ensure that everything arrives in tact and nothing gets lost during transmission.

CRC - CRC stands for Cyclic Redundancy Check. CRC is an error-detecting code used to determine if a block of data has been corrupted. This field is 4 bytes big.
